## 1. 问题清单

**用途**：把所有问题摊在桌面上，从纷乱中获得掌控感。

**使用方法**：
1. 拿出一张纸，写下当前面临的所有问题
2. 每个问题一行，不排序、不评判
3. 写完后，感受"问题被保管在清单里"的轻松感

**关键原则**：
- 不在同一时刻解决多个问题
- 空间上的"远一点儿法则"：问题在清单里，不在你心里
- 时间上的"远一点儿法则"：用一段时间去平摊

---

## 2. 紧急重要模型

**用途**：对问题清单进行分类，找到优先级。

```
        | 重要          | 不重要
--------|--------------|----------
紧急    | 第一象限     | 第三象限
        | 必须处理     | 交给别人
--------|--------------|----------
不紧急  | 第二象限     | 第四象限
        | 优先驾驭★   | 放手
```

**核心**：优先处理"重要不紧急"问题，从紧张的节奏里解脱。

**警惕**：误把紧急当重要——最常见的陷阱。

---

## 3. 价值罗盘

**用途**：从全局视角判断问题的价值和优先级。

**使用方法**：
1. 画出4个维度：家庭、事业、健康、成长
2. 在每个维度标注当前最重要的问题
3. 找到跨维度影响最大的问题 = "问题之王"

---

## 4. 人际生态图

**用途**：可视化你周围的全部人际关系。

**使用方法**：
1. 在中心画自己
2. 按亲疏关系画出周围的人
3. 标注每段关系的质量（信任/紧张/中立）
4. 识别资源与隐患

---

## 5. 关系人图

**用途**：识别与当前问题直接相关的人。

**使用方法**：
1. 列出所有与问题有关的人
2. 标注每个人对问题的影响力和受影响程度
3. 明确"双重身份"：问题管理者 vs 重要关系人
4. 组建问题解决项目团队

**关键**：把自己画进图中，躬身入局。

---

## 6. MECE法则

**用途**：横向展开问题，看到全貌。

**MECE = Mutually Exclusive, Collectively Exhaustive**
- 相互独立：子问题之间不重叠
- 完全穷尽：合在一起覆盖问题全貌

**使用方法**：
1. 将"问题之王"拆分为3-7个子问题
2. 检查：子问题之间有无重叠？
3. 检查：是否遗漏了重要方面？
4. 重复拆分直到可操作

**示例**：
```
问题：项目延期
├── 人员问题（缺人/能力不足）
├── 资源问题（预算/设备不足）
├── 流程问题（审批慢/沟通不畅）
└── 外部问题（需求变更/供应商延迟）
```

## 7. 问题X光片

**用途**：像拍X光一样，让真正的问题浮出水面。

**使用方法**：
1. 写下你认为的问题
2. 区分：这是"观点"还是"事实"？
3. 如果是观点，追问：具体事实是什么？
4. 用数据/事件替代概括性描述

**警示**：
- "团队不配合" → 观点
- "过去2周，3位同事未在约定时间提交材料" → 事实

---

## 8. 1-10标尺法

**用途**：将主观感受量化，让问题可衡量。

**使用方法**：
1. 设定1-10的标尺
2. 1 = 最差/最低，10 = 最好/最高
3. 对每个子问题的现状打分
4. 明确目标分数

**示例**：
- 现状：团队协作效率 → 4分
- 目标：团队协作效率 → 8分
- 差距Δ = 8 - 4 = 4

---

## 9. SMART原则

**用途**：设定有效的目标。

- **S (Specific)** 具体：目标要明确具体，不模糊
- **M (Measurable)** 可量化：有衡量标准
- **A (Achievable)** 可达成：跳一跳够得着
- **R (Relevant)** 相关：与核心关切一致
- **T (Time-bound)** 有时限：有明确的截止时间

**反面例子**："我要变得更好" — 不SMART
**正面例子**："3个月内将客户满意度从70%提升到85%" — SMART

---

## 10. 目标剪枝法

**用途**：大胆剪掉多余目标，聚焦核心。

**使用方法**：
1. 列出所有想达成的目标
2. 问：哪些是"蓝莓枝"（看似茂盛但不结果）？
3. 果断剪掉，保留核心目标
4. 方向永远优于速度

---

## 11. Δ差距量化

**用途**：精准计算现状与目标的差距。

**公式**：
```
Δ = |现状 - 目标|
```

**使用方法**：
1. 用1-10标尺法量化现状
2. 用SMART原则定义目标
3. 计算Δ值
4. Δ越大，问题越"大"，也意味着改变空间越大

---

## 12. 代价评估

**用途**：量化不解决问题的损失，提供行动推力。

**使用方法**：
1. 问：如果这个差距持续1年/3年/5年，会怎样？
2. 从多个维度评估代价：经济、关系、健康、成长
3. 量化代价（金额/时间/情感损失）
4. 这份代价就是你行动的"推力"

---

## 13. P=p-i公式

**用途**：找到解决方案的底层逻辑。

```
Performance表现 = Potential潜能 - Interference干扰
```

**核心洞察**：
- 提升表现有两条路：提升潜能 or 排除干扰
- 排除干扰往往更有效、更快速
- 解决问题 = 排除干扰的过程

---

## 14. KSME四维度分析

**用途**：系统性地排查干扰来源。

| 维度 | 含义 | 典型干扰 | 排除方式 |
|------|------|---------|---------|
| K | Knowledge知识 | "我不知道" | 学习/培训/请教专家 |
| S | Skill技能 | "我不会做" | 练习/辅导/工具辅助 |
| M | Motivation动机 | "我不想做" | 激励/意义感/自主权 |
| E | Environment环境 | "条件不允许" | 调整环境/争取资源/改变规则 |

**使用方法**：
1. 从K/S/M/E四个维度逐一排查
2. 每个维度列出具体干扰项
3. 优先排除"投入小、效果大"的干扰

## 15. 头脑风暴

**用途**：发散思维，穷尽备选方案。

**规则**：
1. 不评判（先不评价好坏）
2. 越多越好（数量优先）
3. 鼓励天马行空
4. 可以在别人想法上叠加

**使用方法**：
1. 明确要解决的问题
2. 限时10-15分钟
3. 每个人依次提出方案
4. 记录所有方案，不做筛选
5. 结束后统一评估

## 16. 6顶思考帽

**用途**：有序地分析方案，达成共识。

| 颜色 | 视角 | 关注点 |
|------|------|--------|
| ⬜ 白帽 | 数据 | 事实和数据是什么？ |
| 🔴 红帽 | 情感 | 直觉和感受是什么？ |
| ⬛ 黑帽 | 风险 | 有什么潜在问题？ |
| 🟡 黄帽 | 价值 | 有什么好处和机会？ |
| 🟢 绿帽 | 创新 | 还有没有别的办法？ |
| 🔵 蓝帽 | 控制 | 我们该怎么组织思考？ |

**使用方法**：
1. 所有人同时戴同一颜色的帽子
2. 按顺序切换（建议：蓝→白→黄→黑→绿→红→蓝）
3. 每顶帽子2-5分钟
4. 蓝帽负责总结决策

## 17. 5W2H行动计划

**用途**：将方案落到可执行的行动。

| 维度 | 问题 | 示例 |
|------|------|------|
| What | 做什么事？ | 完成项目方案书 |
| Who | 谁负责？ | 张三 |
| When | 何时完成？ | 本周五17:00前 |
| Where | 在哪里？ | 线上协作文档 |
| Why | 为什么做？ | 为下周评审做准备 |
| How | 怎么做？ | 按模板填写+内部评审 |
| How much | 需要多少资源？ | 2人×3小时 |

**原则**：凡是需要落实的行动，无论多小，都列在表格中。不能靠记忆，不要口头传递。

## 18. PDCA循环

**用途**：持续迭代，确保问题真正解决。

```
Plan(计划) → Do(执行) → Check(检查) → Act(处理)
    ↑                                    |
    └────────────────────────────────────┘
```

**Plan**：制定行动计划
**Do**：执行计划
**Check**：检查执行情况（检查的是自己的执行，不是催促他人）
**Act**：根据检查结果调整，进入下一轮循环

## 19. 情绪ABC理论

**用途**：理解情绪来源，管理情绪。

- **A (Activating Event)** 触发事件
- **B (Belief)** 信念/想法 → 这是关键变量
- **C (Consequence)** 情绪后果

**核心洞察**：决定你情绪的不是事件本身，而是你对事件的解读（信念）。

**应用**：改变B（信念），就能改变C（情绪后果）。

---

## 20. 金色信念/灰色信念

**用途**：用积极信念替代消极信念。

- **灰色信念**："我不行""这不可能""又是我的错"
- **金色信念**："我可以试试""有其他可能""这是我的机会"

**操作**：识别灰色信念 → 替换为金色信念 → 强化练习

---

## 21. 影响圈/关注圈

**用途**：聚焦可控范围，减少内耗。

```
┌─────────────────────────────┐
│       关注圈                 │
│   ┌───────────────────┐     │
│   │    影响圈          │     │
│   │   我能改变的       │     │
│   └───────────────────┘     │
│   我关心但无法直接控制的       │
└─────────────────────────────┘
```

**核心**：从影响圈出发，让核心子问题浮出水面。专注你能改变的事。

## 22. 复盘报告模板

**用途**：第八步复盘专用，系统回顾整个问题解决过程。

**使用方法**：
1. 逐个步骤回顾，评价每步的难易度和效果
2. 提炼成功经验（做对了什么）
3. 识别问题和改进点（哪里可以更好）
4. 提炼可迁移经验（能否复用到其他问题）
5. 制定后续跟进计划

**复盘核心问题**：
- 目标是否达成了？如果没有，差距在哪里？
- 哪一步最有价值？哪一步最困难？
- 有什么是我之前没看到的？
- 这个经验可以用在哪里？

**AI引导语示例**：
- "回顾整个流程，你觉得自己在哪一步收获最大？"
- "如果重新来过，你会在哪一步做不同的选择？"
- "这个方法论，你觉得还能用在什么问题上？"
