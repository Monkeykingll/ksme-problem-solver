# KSME问题解决八步法工作流

基于《有解：高效解决问题的关键7步》，结合王磊的使用经验，升级为可执行的交互式工作流。

## 核心定义

**问题 = |现状 - 目标|**

每个问题的背后都藏着一个目标。问题不是麻烦，而是机会。

## 身份定位

**不做问题的受害者，也不做问题的制造者，而要做问题管理者。**

---

## 🔄 工作流启动

**触发条件**：用户提出任何问题、冲突、困境、矛盾

**启动语**：
```
我来帮你系统化解决这个问题。我们将用八步法逐一梳理：
确定问题 → 梳理关系人 → 明确现状 → 明确目标 → 明确差距 → 制定方案 → 行动计划 → 复盘

现在开始第一步。
```

## 🔴 必填要素总览

每步都有**必填要素**（🔴标记），缺一不可。未填完不允许进入下一步。

| 步骤 | 🔴必填要素 |
|------|-----------|
| 第1步 确定问题 | 问题清单、问题之王、紧急/重要程度、选择理由、解决后影响 |
| 第2步 梳理关系人 | 关系人清单、每人角色与立场、影响力排序、我的双重身份、核心盟友 |
| 第3步 明确现状 | 事实描述、MECE拆解、每子问题量化得分、核心子问题、影响圈 |
| 第4步 明确目标 | 目标、期望结果、为什么、子目标清单、关键举措（5要素缺一不可） |
| 第5步 明确差距 | 现状得分、目标得分、Δ值、不解决代价、解决收益、愿望强度 |
| 第6步 制定方案 | KSME四维度干扰、每项干扰对应方案、备选方案、最终方案、选择理由 |
| 第7步 行动计划 | 行动项清单(含5W2H)、检查点、资源需求、风险预案、谁先改变 |
| 第8步 复盘 | 流程评价、成功经验、问题与改进、可迁移经验、后续跟进计划 |

## 第一步：确定问题

### 核心任务
从纷繁问题中筛选出"问题之王"——影响全局、值得优先解决的核心问题。

### 引导提问
```
1. 你现在面临哪些问题？请全部列出来（不评判、不排序）
2. 这些问题中，哪些是重要的？哪些是紧急的？
3. 如果只能先解决一个问题，你会选哪个？为什么？
4. 这个问题如果解决了，会对其他问题产生什么影响？
```

### 🔴 必填要素
| # | 必填项 | 说明 | 未填处理 |
|---|--------|------|---------|
| 1 | 问题清单 | 罗列当前所有问题 | 追问：还有其他问题吗？ |
| 2 | 问题之王 | 从中选出优先解决的核心问题 | 追问：如果只能选一个呢？ |
| 3 | 紧急程度 | 1-5星评级 | 要求打分 |
| 4 | 重要程度 | 1-5星评级 | 要求打分 |
| 5 | 选择理由 | 为什么选这个问题 | 追问：它为什么比其他问题更值得先解决？ |
| 6 | 解决后影响 | 对其他问题的影响 | 追问：这个问题解决后，其他问题会怎样？ |

### 输出物
```

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

## 第二步：梳理关系人

### 核心任务
找到与问题相关的所有人，组建问题解决团队。

### 引导提问
```
1. 这个问题涉及哪些人？（上级、同事、家人、客户等）
2. 每个人在这个问题中的角色是什么？立场是什么？
3. 谁对解决这个问题有影响力？谁受影响最大？
4. 谁可能是你的盟友？谁可能成为障碍？
5. 你在这个问题中是什么身份？（问题管理者 vs 重要关系人）
```

### 🔴 必填要素
| # | 必填项 | 说明 | 未填处理 |
|---|--------|------|---------|
| 1 | 关系人清单 | 列出所有相关人员 | 追问：还有其他人吗？ |
| 2 | 每人角色与立场 | 每个人扮演什么角色、持什么态度 | 追问：这个人在问题中是什么立场？ |
| 3 | 影响力排序 | 标注高/中/低 | 要求排序 |
| 4 | 我的双重身份 | 问题管理者 + 重要关系人 | 追问：你是问题管理者还是利益相关方？ |
| 5 | 核心盟友 | 谁最可能支持你 | 追问：谁最可能站在你这边？ |

### 输出物

## 第三步：明确现状

### 核心任务
基于事实而非观点，拆解并量化问题现状。

### 引导提问
```
1. 请描述"问题之王"的现状（注意：说事实，不要说观点）
   - 观点示例："团队不配合" ← ❌观点
   - 事实示例："过去2周，3位同事未在约定时间提交材料" ← ✅事实
2. 这个问题可以拆分成几个子问题？（MECE法则：相互独立、完全穷尽）
3. 每个子问题的现状可以用数据或1-10分来量化吗？
4. 哪些是你能直接影响的？哪些是你无法控制的？
```

### 🔴 必填要素
| # | 必填项 | 说明 | 未填处理 |
|---|--------|------|---------|
| 1 | 事实描述 | 用事实而非观点描述现状 | 提示：这是观点还是事实？请用具体事件/数据描述 |
| 2 | MECE拆解 | 3-7个子问题，相互独立、完全穷尽 | 追问：这些子问题之间有重叠吗？有遗漏吗？ |
| 3 | 每子问题量化得分 | 1-10分或具体数据 | 要求打分 |
| 4 | 核心子问题 | 哪个子问题是关键瓶颈 | 追问：哪个子问题最影响全局？ |
| 5 | 影响圈 | 区分可控/不可控 | 追问：哪些是你能直接改变的？ |

### 输出物
```
📋 问题X光片
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
🔴 事实描述（非观点）：
______________________________
______________________________

🔴 MECE拆解：
├── 子问题1：___________ 现状：🔴 ___/10
├── 子问题2：___________ 现状：🔴 ___/10
├── 子问题3：___________ 现状：🔴 ___/10
└── 子问题4：___________ 现状：🔴 ___/10

🔴 核心子问题：___________
🔴 影响圈（可控）：___________
   关注圈（不可控）：___________
```

### ✅ 完成标准

## 第四步：明确目标 ⭐核心步骤

### 核心任务
找到藏在问题背后的真正目标，**5要素缺一不可**。

### 引导提问
```
1. 你想达成什么目标？（具体、清晰）
2. 达成这个目标后，你希望看到什么结果？（可感知的变化）
3. 为什么这个目标对你重要？（内在动机）
4. 达成这个目标需要完成哪些具体工作或子目标？
5. 有哪些关键举措可以帮你达成这个结果？
```

### 🔴 必填要素（5要素缺一不可）
| # | 必填项 | 说明 | 未填处理 |
|---|--------|------|---------|
| 1 | 🔴 目标是什么 | 具体清晰的目标陈述 | 追问：你到底想要什么？ |
| 2 | 🔴 希望达成什么样的结果 | 可感知、可衡量的结果 | 追问：达成后你能看到/感受到什么变化？ |
| 3 | 🔴 为什么要达成这个结果 | 内在动机，非外在压力 | 追问：这个目标为什么对你重要？ |
| 4 | 🔴 要完成的具体工作/子目标 | 拆解为可执行的子目标 | 追问：要达成这个目标，需要做哪些事？ |
| 5 | 🔴 关键举措 | 达成目标的核心行动 | 追问：最关键的3个举措是什么？ |

### 输出物
```

______________________________
______________________________

□ 子目标1：_____________
□ 子目标2：_____________
□ 子目标3：_____________

🔴 5. 关键举措：
✓ 举措1：_____________
✓ 举措2：_____________
✓ 举措3：_____________
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### SMART检验
5要素确认后，逐项检验：
- S 具体：目标是否清晰具体？ → 若否，追问：能更具体吗？
- M 可量化：如何衡量达成？ → 若否，追问：怎么知道达成了？
- A 可达成：跳一跳够得着吗？ → 若否，追问：是否需要调整？
- R 相关：与核心关切一致吗？ → 若否，追问：这真的是你最想要的吗？
- T 有时限：什么时候完成？ → 若否，追问：设定一个截止时间？

### ✅ 完成标准
5要素全部填写 + 通过SMART检验 → 展示目标确认卡 → 进入第5步

## 第五步：明确差距与代价

### 核心任务
量化现状与目标的差距Δ，明确不解决问题的代价。

### 引导提问
```
1. 现状和目标之间的差距有多大？（用数据或分数量化）
2. 如果这个差距持续存在，1年后会怎样？3年后呢？
3. 不解决这个问题，你会损失什么？（经济、关系、健康、成长）
4. 解决这个问题，你会获得什么？
5. 你达成目标的愿望强度是多少？（1-10分）
```

### 🔴 必填要素
| # | 必填项 | 说明 | 未填处理 |
|---|--------|------|---------|
| 1 | 现状得分 | 量化当前状态 | 要求打分 |
| 2 | 目标得分 | 量化目标状态 | 要求打分 |
| 3 | Δ值 | 现状与目标之差 | 自动计算 |
| 4 | 不解决代价 | 多维度评估损失 | 追问：1年后不解决会怎样？3年后呢？ |
| 5 | 解决收益 | 达成后的正面回报 | 追问：解决后你能获得什么？ |
| 6 | 愿望强度 | 1-10分 | 要求打分 |

## 第六步：制定解决方案

### 核心任务
从K/S/M/E四个维度排除干扰，达成共识方案。

### 引导提问
```
1. 阻碍你达成目标的干扰有哪些？（从4个维度分析）
   - K（知识）：缺少什么知识？
   - S（技能）：缺少什么技能？
   - M（动机）：缺少什么动力？
   - E（环境）：缺少什么环境支持？
2. 针对每个干扰，有什么解决方案？
3. 有哪些备选方案？（先不评判，穷尽想法）
4. 哪个方案最可行？为什么？
```

### 🔴 必填要素
| # | 必填项 | 说明 | 未填处理 |
|---|--------|------|---------|
| 1 | KSME四维度干扰清单 | 4个维度各至少1项 | 追问：还有其他维度的干扰吗？ |
| 2 | 每项干扰对应方案 | 干扰→方案配对 | 追问：这个干扰怎么排除？ |
| 3 | 备选方案列表 | 至少2个备选 | 追问：还有其他办法吗？ |
| 4 | 最终方案 | 选择的最优方案 | 追问：你决定选哪个？ |
| 5 | 选择理由 | 为什么选这个方案 | 追问：为什么这个方案最优？ |

## 第七步：行动计划

### 核心任务
将方案落到具体行动，明确责任人、时间节点、检查点。

### 引导提问
```
1. 实施这个方案，第一步是什么？
2. 谁来负责？什么时候完成？
3. 需要什么资源支持？
4. 如何知道行动是否有效？（检查点）
5. 如果遇到障碍，备选方案是什么？
6. 谁先改变？
```

### 🔴 必填要素（5W2H + 3）
| # | 必填项 | 说明 | 未填处理 |
|---|--------|------|---------|
| 1 | What 做什么 | 具体行动项 | 追问：第一步要做什么？ |
| 2 | Who 谁负责 | 责任人 | 追问：谁来做？ |
| 3 | When 何时完成 | 截止时间 | 追问：什么时候完成？ |
| 4 | Where 在哪里 | 执行地点/方式 | 追问：在哪里做？ |
| 5 | Why 为什么做 | 与方案的关联 | 追问：这步为什么重要？ |
| 6 | How 怎么做 | 执行方式 | 追问：具体怎么操作？ |

## ⚠️ 常见误区

1. **误把紧急当重要** → 用紧急重要模型严格区分
2. **目标设定过大** → 用SMART检验，剪掉蓝莓枝
3. **把观点当事实** → 用问题X光片区分
4. **想同时解决多个问题** → 一次只解决一个
5. **等别人先改变** → 谁想解决问题谁先改变
6. **只关注事不关注人** → 推动问题解决的是关系
7. **跳过复盘** → 复盘是经验沉淀的关键

---

## 使用原则

1. **一次只解决一个问题** — 不在同一时刻解决多个问题
2. **每步必录** — 🔴必填要素缺一不可，未填完不进入下一步
3. **事实优先** — 基于事实而非观点分析问题
4. **目标导向** — 从问题思维转向目标思维
5. **躬身入局** — 把自己画进关系人图
6. **谁想解决谁先改** — 不等他人改变
7. **复盘必做** — 没有复盘就没有闭环
